# BremLogic iOS Local Install

This project now includes a private iOS shell built with Capacitor so BremLogic can run as a native app on your own iPhone without a public App Store release.

## What Was Added

- `capacitor.config.ts`
  - Native app id: `com.bremlogic.signalsbot`
  - App name: `BremLogic`
  - Live app URL: `https://app.bremlogic.com/signals-bot`
- `ios/App`
  - Xcode project generated by Capacitor
- `hooks/usePhantomBrowserSdkWallet.ts`
  - Phantom mobile callback handling now recognizes the native `bremlogic://` return path

## Important Caveat

The iOS shell currently loads the live hosted app from `app.bremlogic.com`.

That means:

- Native shell changes work immediately once you install the iOS app locally.
- Web-app changes in this repo only appear inside the iOS app after they are deployed to the live site.

If you want to test unpublished web changes on your phone, temporarily point `server.url` in [capacitor.config.ts](/Users/lyrastudio/Documents/BremBot/crypto-signal-dashboard/capacitor.config.ts) to a LAN URL such as:

```ts
server: {
  url: "http://192.168.1.25:3000/signals-bot",
  cleartext: true,
}
```

Then run the Next app locally with `npm run dev` and make sure the iPhone is on the same Wi‑Fi network.

## Local Xcode Install

1. Install Xcode on your Mac.
2. Open Terminal in `/Users/lyrastudio/Documents/BremBot/crypto-signal-dashboard`.
3. Run:

```bash
npm install
npm run cap:sync:ios
npm run cap:open:ios
```

4. In Xcode, select the `App` project.
5. In `Signing & Capabilities`:
   - choose your Apple ID team
   - keep the bundle id as `com.bremlogic.signalsbot`, or change it if that id is already in use
6. Plug in your iPhone or select it from the device menu.
7. Press the Run button in Xcode.
8. If iOS warns about an untrusted developer certificate:
   - open `Settings > General > VPN & Device Management`
   - trust your developer certificate
   - reopen BremLogic

## Wallet Return Path

The generated iOS app registers the custom URL scheme:

- `bremlogic://`

That is the scheme the native shell uses for wallet round-trip handling instead of relying on the mobile browser.

## Useful Commands

```bash
npm run cap:sync:ios
npm run cap:open:ios
```

If you change native project files or Capacitor config, run `npm run cap:sync:ios` again before reopening Xcode.
